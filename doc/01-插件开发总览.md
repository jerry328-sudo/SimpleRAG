# Obsidian 插件开发总览

官方文档把 Obsidian 扩展方式分成三类：

1. 官方开发文档：定义开发流程、发布规则、API 范围与规范。
2. 示例插件仓库：给可运行的工程结构。
3. API 类型定义仓库：给完整签名和版本边界。

如果把它们结合起来，插件开发链路就是：

1. 按官方教程创建工程并运行 `npm run dev`。
2. 通过 `.obsidian/plugins/<id>/` 加载 `main.js`。
3. 在 `main.ts` 里注册命令、UI、Vault 操作。
4. 调试通过后整理版本与发布信息。
5. 通过 GitHub Release 进行分发。
6. 可选但建议提交官方社区插件目录。

## 1. 官方入口

- 开发文档首页： https://docs.obsidian.md/Home
- 从零教程： https://docs.obsidian.md/Plugins/Getting+started/Build+a+plugin
- 示例插件： https://github.com/obsidianmd/obsidian-sample-plugin
- API 类型定义： https://github.com/obsidianmd/obsidian-api

## 2. 插件最核心事实

- 插件默认使用 TypeScript 开发。  
- 编译后入口是 `main.js`，Obsidian 直接加载这个文件。
- 清单是 `manifest.json`，决定插件显示名、id、版本、最低兼容版本、是否桌面专用。
- 官方生态依赖 `obsidian` API 类型，而不是直接捆绑宿主模块。

## 3. 一个最小开发目录

```text
.obsidian/
└─ plugins/
   └─ your-plugin-id/
      ├─ main.js
      ├─ manifest.json
      ├─ styles.css
      └─ (optional) versions.json
```

## 4. 从官方教程到实战的对应关系

- 文档说“配置开发环境”对应：Node、Git、`npm install`、`npm run dev`。
- 文档说“加载插件”对应：放到 `.obsidian/plugins/` 目录并在插件设置面板开启。
- 文档说“开始开发”对应：修改 `manifest.json`、`src/main.ts`。
- 文档说“发布”对应：发布 release + 可选提交 `community-plugins.json`。

## 5. 文档查阅建议

按任务查文档可显著提高效率：

1. 环境搭建：先看 `Build a plugin` 和 sample plugin README。
2. 写具体功能：先看界面/设置/命令文档，再查 `obsidian.d.ts`。
3. 发布与审核：先看 `Plugin guidelines` 和 `Submit your plugin`。
4. 兼容性与性能：看移动开发和加载优化相关文档。

## 6. 你会在后续文档里得到什么

- `Build a plugin` 的完整本地开发操作法。
- `obsidian-sample-plugin` 的每个核心文件用途。
- 典型 API 使用样式与推荐写法。
- 审核和发布前的检查项清单。
- API 仓库中查签名和版本门槛的方法。
