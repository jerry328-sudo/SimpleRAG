# 最小可复制脚手架

这一篇的目的不是替代官方 sample plugin，而是把官方 sample plugin 的最小核心骨架直接写成可复制的 Markdown，方便你在离线阅读时也能手工搭一个基础插件工程。

注意：

- 这份脚手架结构和思路参考官方 `obsidian-sample-plugin`
- 它能让你从零写出一个最小可运行插件
- 但它不能替代联网安装依赖这件事本身

## 1. 目录结构

```text
hello-world/
├─ manifest.json
├─ package.json
├─ tsconfig.json
├─ esbuild.config.mjs
├─ version-bump.mjs
├─ versions.json
├─ styles.css
└─ src/
   ├─ main.ts
   └─ settings.ts
```

```json
{
  "name": "hello-world",
  "version": "0.0.1",
  "description": "Hello World plugin for Obsidian",
  "main": "main.js",
  "type": "module",
  "scripts": {
    "dev": "node esbuild.config.mjs",
    "build": "tsc -noEmit -skipLibCheck && node esbuild.config.mjs production",

  },
  "devDependencies": {
    "@types/node": "^16.11.6",
    "esbuild": "0.25.5",
    "typescript": "^5.8.3"
  },
  "dependencies": {
    "obsidian": "latest"
  }
}
```

### 3.1 为什么这里基本照着官方 sample plugin 写

- `obsidian` 作为依赖提供类型与 API 声明
- `esbuild` 负责把 `src/main.ts` bundle 成 `main.js`
- `typescript` 负责类型检查
- `version` 脚本用来同步版本号

## 5. `esbuild.config.mjs`

```js
import esbuild from "esbuild";
import process from "process";
import { builtinModules } from "node:module";

const banner =
  "/* THIS IS A GENERATED/BUNDLED FILE BY ESBUILD */";

const prod = process.argv[2] === "production";

const context = await esbuild.context({
  banner: {
    js: banner
  },
  entryPoints: ["src/main.ts"],
  bundle: true,
  external: [
    "obsidian",
    "electron",
    "@codemirror/autocomplete",
    "@codemirror/collab",
    "@codemirror/commands",
    "@codemirror/language",
    "@codemirror/lint",
    "@codemirror/search",
    "@codemirror/state",
    "@codemirror/view",
    "@lezer/common",
    "@lezer/highlight",
    "@lezer/lr",
    ...builtinModules
  ],
  format: "cjs",
  target: "es2018",
  logLevel: "info",
  sourcemap: prod ? false : "inline",
  treeShaking: true,
  outfile: "main.js",
  minify: prod
});

if (prod) {
  await context.rebuild();
  process.exit(0);
} else {
  await context.watch();
}
```

## 6. `version-bump.mjs`

```js
import { readFileSync, writeFileSync } from "fs";

const targetVersion = process.env.npm_package_version;

const manifest = JSON.parse(readFileSync("manifest.json", "utf8"));
const { minAppVersion } = manifest;

manifest.version = targetVersion;
writeFileSync("manifest.json", JSON.stringify(manifest, null, "\t"));

const versions = JSON.parse(readFileSync("versions.json", "utf8"));

if (!Object.values(versions).includes(minAppVersion)) {
  versions[targetVersion] = minAppVersion;
  writeFileSync("versions.json", JSON.stringify(versions, null, "\t"));
}
```

## 7. `versions.json`


## 9. `src/settings.ts`

```ts
import { App, PluginSettingTab, Setting } from "obsidian";
import HelloWorldPlugin from "./main";

export interface HelloWorldSettings {
  greetingText: string;
}

export const DEFAULT_SETTINGS: HelloWorldSettings = {
  greetingText: "Hello, world!"
};

export class HelloWorldSettingTab extends PluginSettingTab {
  plugin: HelloWorldPlugin;

  constructor(app: App, plugin: HelloWorldPlugin) {
    super(app, plugin);
    this.plugin = plugin;
  }

  display(): void {
    const { containerEl } = this;
    containerEl.empty();

    new Setting(containerEl)
      .setName("Greeting text")
      .setDesc("The message shown when the ribbon icon is clicked.")
      .addText((text) =>
        text
          .setPlaceholder("Hello, world!")
          .setValue(this.plugin.settings.greetingText)
          .onChange(async (value) => {
            this.plugin.settings.greetingText = value;
            await this.plugin.saveSettings();
          })
      );
  }
}
```

## 10. `src/main.ts`

```ts
import { Notice, Plugin } from "obsidian";
import {
  DEFAULT_SETTINGS,
  HelloWorldSettingTab,
  HelloWorldSettings
} from "./settings";

export default class HelloWorldPlugin extends Plugin {
  settings: HelloWorldSettings;

  async onload() {
    await this.loadSettings();

    this.addRibbonIcon("dice", "Say hello", () => {
      new Notice(this.settings.greetingText);
    });

    this.addCommand({
      id: "say-hello",
      name: "Say hello",
      callback: () => {
        new Notice(this.settings.greetingText);
      }
    });

    this.addSettingTab(new HelloWorldSettingTab(this.app, this));
  }

  async loadSettings() {
    this.settings = Object.assign({}, DEFAULT_SETTINGS, await this.loadData());
  }

  async saveSettings() {
    await this.saveData(this.settings);
  }
}
```

## 11. 放到哪里

你的目录必须是：

```text
<vault>/.obsidian/plugins/hello-world/
```

这里的 `hello-world` 要与 `manifest.json` 里的 `id` 一致。

## 12. 第一次启动

在该目录中执行：

```bash
npm install
npm run dev
```

然后在 Obsidian 中：

1. 打开 `Settings`
2. 进入 `Community plugins`
3. 打开社区插件开关
4. 启用 `Hello World`

## 13. 看到什么算成功

成功后你应该至少看到两件事：

1. 左侧 Ribbon 上出现一个骰子图标
2. 命令面板里能找到 `Say hello`

## 14. 这个脚手架覆盖了哪些官方能力

- `Plugin` 生命周期
- `addRibbonIcon`
- `addCommand`
- `loadData` / `saveData`
- `PluginSettingTab`
- `Setting`
- `Notice`
- `esbuild` 构建到 `main.js`
