# 从零开始构建插件

该篇按官方教程重建“能跑”的开发环境，目标是本地能安装并加载第一个插件。

## 1. 准备工作

### 1.1 必备工具

- Git
- Node.js（官方样例模板和当前生态通常使用 Node 16+，推荐 18 LTS）
- VS Code 等编辑器

### 1.2 先准备开发 Vault

官方教程建议使用独立开发 vault，避免误操作主库笔记。

```text
<dev-vault>/
└─ .obsidian/
   └─ plugins/
```

## 2. 下载官方样例并初始化

官方最推荐的起点是官方样例仓库：

- https://github.com/obsidianmd/obsidian-sample-plugin

常见初始化流程：

1. 将样例仓库放入 `.obsidian/plugins/` 下
2. 修改为自己的仓库名
3. 安装依赖
4. 运行 `npm run dev`

PowerShell 示例：

```powershell
cd .obsidian\plugins
git clone https://github.com/obsidianmd/obsidian-sample-plugin.git
cd obsidian-sample-plugin
npm install
npm run dev
```

## 3. 运行结构解读

`npm run dev` 会把 TypeScript 编译为 `main.js`，而 Obsidian 只会加载 `main.js`。

- 你改的是 `src/main.ts`
- Obsidian 运行的是项目根目录下的 `main.js`

若插件无响应，先检查：

- `npm run dev` 是否在运行
- `main.js` 是否有生成
- 插件是否放到正确目录 `.obsidian/plugins/<id>/`

## 4. 在 Obsidian 内加载

1. 打开 `Settings`
2. 进入 `Community plugins`
3. 打开 `Turn on community plugins`
4. 在 `Installed plugins` 里启用你的插件

提示：首次加载时如果 `manifest.json` 有变更，建议重启 Obsidian。

## 5. 第一次修改你应该做的事

### 5.1 修改 `manifest.json`

至少改这几个字段：

- `id`
- `name`
- `description`
- `version`
- `minAppVersion`
- `author`

`id` 最好和目录名一致。例如：

```text
.obsidian/plugins/hello-world/manifest.json
```

### 5.2 触发一次最小可见动作

在 `src/main.ts` 注册 Ribbon 图标，确认开发链路打通：

```ts
import { Notice, Plugin } from 'obsidian';

export default class HelloWorldPlugin extends Plugin {
  async onload() {
    this.addRibbonIcon('dice', 'Hello', () => {
      new Notice('Hello, Obsidian!');
    });
  }
}
```

如果能看到图标并弹出通知，说明环境是正常的。

## 6. 修改后重载方式

- 常用方式：在插件列表里关闭再开启插件。
- 改 `manifest.json`：建议重启 Obsidian。
- 可选工具：开发时装一个 Hot-Reload 插件提升效率。

## 7. sample plugin 的价值

官方样例本质上就是一套最小但完整的工程框架。它能直接告诉你：

- 插件入口怎么写
- 设置页怎么拆
- 注册命令、事件、Markdown 后处理怎么做
- `package.json`/`tsconfig.json`/`esbuild.config.mjs` 的默认形态

因此建议你把样例当作“模板 + 结构参考”，不是直接照抄功能代码。

## 8. 新手常见问题

- 看不到插件：确认路径与 `id` 是否一致、是否有 `manifest.json`、`main.js` 是否生成。
- 看到了但命令没生效：确认重载；确认 `addCommand`/`addRibbonIcon` 已在 `onload` 调用。
- 开发很慢：检查 `npm run dev` 是否已在 watch 模式执行。
